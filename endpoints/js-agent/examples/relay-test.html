OpenNHP JavaScript SDK Demo

The OpenNHP JavaScript SDK lets a browser app “knock” an nhp-server for zero-trust access before redirecting the user to a protected resource. 🔧 Integration Steps
1. Create the agent. Use transport: 'relay' in browsers because web pages cannot send UDP packets directly.
2. Set identity. Pass the current app user, device, and organization identifiers into the knock payload.
3. Add server. Pin the nhp-server public key your app trusts.
4. Knock resource. Ask for a resource/service pair and redirect only after a successful ACK.
⌨ Minimal Browser Code
import { NHPAgent } from '@opennhp/agent';

const agent = new NHPAgent({
  transport: 'relay',
  relayUrl: 'https://relay.example.com/relay',
  privateKey: agentPrivateKey,
});

await agent.init();
agent.setIdentity({
  userId: currentUser.email,
  deviceId: browserDeviceId,
  organizationId: 'opennhp.org',
});
agent.addServer({ publicKey: serverPublicKey });

const result = await agent.knockResource({
  resourceId: 'demo',
  serviceId: 'example',
});

if (result.success) {
  window.location.href =
    `https://${Object.values(result.resourceHosts)[0]}`;
}

This public demo intentionally uses demo keys. Production apps should issue per-user or per-device keys and bundle crypto dependencies with the application.
